Vue, React или Angular

React – первое и основное что мы должны заметить, react это библиотека, а не фреймворк, как vue и angular. То есть это библиотека, которая пользуется нашим кодом, а вот Фреймворк вызывает наш код. То есть реакт – это набор инструментов. React одна из самых популярных библиотек, достаточно прост в изучении, созданные компоненты могут быть легко изменены и использованы в новых проектах. Благодаря изоморфному подходу рендеринг страниц производится быстрее, сайт гораздо лучше индексируют поисковики. Также в React представлена Virtual Document Object Model (VDOM) – способ взаимодействия с html, древовидная структура данных которая призвана гораздо быстрее рендерить элементы. Если выбирать между React, Vue и Angular, то React это золотая середина. Превосходит vue в разнообразии и изобилии доступных средств отрисовки. Чуть сложнее в изучении чем Vue, но гораздо легче чем Angular.

Плюсы

  • Легко изучать
  • Очень быстрый благодаря VDOM
  • Простой для тестирования и многократно используемый код
  • Простой переход между версиями

Минусы

  • Есть некоторые проблемы с разницей архитектур при обновлениях
  • JSX не лучшая идея для разработки веб приложений, слишком сложный

Vue – js фреймворк, который отлично подходит для сложных одностраничных приложений. Достаточно прост в изучении, отличная документация на русском лежит на официальном сайте, что является большим преимуществом. Очень легкий, хорошо расширяемый-кастомизируемый. Но также бывают некоторые проблемы, которые требуют упростить данные для избегания головной боли.

Плюсы

  • Производительность (Также есть VDOM)
  • Простой в изучении
  • Подробная документация на русском
  • Небольшой размер

Минусы

  • Относительно небольшое комьюнити
  • Относительное небольшая доля рынка по сравнению с React и Angular
  • Могут возникнуть проблемы интеграции в огромные проекты

Angular – это oper source фреймворк, достаточно популярный фреймворк. Самым большое преимущество – это большое комьюнити, большое количество людей с кем можно посоветоваться или задать вопросы касательно ангулара. Достаточно простой в чтении код, но при этом достаточно сложный в изучении. Angular очень хорош для огромных проектов, либо же для сложных SPA (single page application). Модульность данного фреймворка тоже можно записывать в плюсы. Многие популярные интернет сообщества заявляют о его простоте тестирования, а разбивка на модули позволяет загружать только нужные службы и эффективно делать автоматическое тестирование. Конечно из минусов сюда можно включить достаточную посредственную документацию, из которой без живого примера сложно что-то понять. «Ванильный» Angular необходимо сразу же подстраивать под себя, отключая некоторые ненужные пакеты, иначе можно получить здоровенные модули, от которых будет рябить в глазах. Достаточной высокий порог входа, нужно потратить гораздо больше времени на изучение и понимание.

Плюсы

  • Универсальность
  • Большое комьюнити
  • Хорош для огромных проектов
  • Модульность
  • Простое тестирование

Минусы

  • Достаточно сложное изучение (большое разнообразие структур)
  • Без подключения changedetectionstrategy очень медленная производительность
  • Высокий порог входа
